四.可能存在的错误

配置hexo博客时报错TypeError: Cannot read properties of null (reading ‘utcOffset‘)
答：在Hexo博客的_config.yml配置文件中，如果timezone设置不正确，如误设为Asia/Beijing，可能会导致问题。正确的中国时区应设为Asia/Harbin,Asia/Shanghai,Asia/Chongqing,Asia/Urumqi或Asia/Kashgar之一。修改这个设置后，保存并重新上传博客即可解决报错。

五.vscode中无法使用连串命令

vscode中无法使用hexo c && hexo g && hexo d # hexo d 表示执行部署命令，最终结果改为使用hexo c ; hexo g ; hexo d # hexo d 表示执行部署
首先提示：标记“&&”不是此版本中的有效语句分隔符。
将“&&”改为分号即可，分号为vscode有效分隔符
后又提示：无法加载文件 C:\Users\clj31\AppData\Roaming\npm\hexo.ps1，因为在此系统上禁止运行脚本。
核心是power shell的安全策略，将 nrm 命令视为了不安全脚本，不允许执行。只需要放开权限就行。
那么问题来了，如何开放权限呢
第一步，win+r，运行powershell
第二步，在控制台中输入Start-Process powershell -Verb runAs，这一步是使用管理员权限运行
第三步，输入set-ExecutionPolicy RemoteSigned改变执行策略
